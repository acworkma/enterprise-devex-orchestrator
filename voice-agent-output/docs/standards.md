## Naming Convention

All resource names follow the **Azure Cloud Adoption Framework (CAF)** pattern:

```
{resourcePrefix}-{workload}-{environment}-{region}
```

### Naming Rules

- **Resource Group:** `rg-{workload}-{env}-{region}`
- **Key Vault:** `kv-{workload}-{env}-{region}` (3-24 chars, alphanumeric + hyphens)
- **Storage Account:** `st{workload}{env}{region}` (3-24 chars, lowercase alphanumeric only)
- **Container Registry:** `cr{workload}{env}{region}` (5-50 chars, alphanumeric only)
- **Container App:** `ca-{workload}-{env}-{region}` (2-32 chars)
- **Log Analytics:** `law-{workload}-{env}-{region}`
- **Managed Identity:** `id-{workload}-{env}-{region}`

## Tagging Standard

All Azure resources **must** include enterprise-mandated tags for cost management,
governance, and operational accountability.

### Required Tags

| Tag | Description | Example |
|---|---|---|
| `project` | Project or workload identifier matching the resource group naming | `my-api-project` |
| `environment` | Deployment environment (dev, staging, prod) | `dev` |
| `costCenter` | Financial cost center for chargeback and billing | `ENG-001` |
| `owner` | Email of the team or individual responsible for this resource | `platform-team@contoso.com` |
| `managedBy` | Tool or process that manages this resource lifecycle | `bicep` |
| `createdBy` | Generator or agent that created this resource definition | `enterprise-devex-orchestrator` |
| `dataSensitivity` | Data classification level per enterprise data policy | `confidential` |

### Optional Tags (Recommended)

| Tag | Description | Example |
|---|---|---|
| `complianceScope` | Applicable compliance framework (general, hipaa, soc2, fedramp) | `soc2` |
| `department` | Business department that owns this workload | `engineering` |
| `team` | Specific team within the department | `platform-engineering` |
| `application` | Application the resource belongs to (for multi-app groups) | `order-processing` |
| `criticality` | Business criticality level for SLA and incident prioritization | `high` |
| `createdDate` | ISO 8601 date when the resource definition was generated | `2025-03-07` |

### Tag Enforcement

- The **Governance Reviewer Agent** validates all generated Bicep templates
  against the tagging standard before scaffolding.
- Missing required tags will result in a **FAIL** governance status.
- Invalid tag values (e.g., malformed email for `owner`) produce warnings.

### Customization

Edit `standards.yaml` in the project root to customize:
- Default cost center, owner, and data sensitivity
- Additional custom required tags
- Department and team defaults
