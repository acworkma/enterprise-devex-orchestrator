## Runbooks

### Container App Restart Alert
- **Severity:** 2 (Warning)
- **Threshold:** > 3 restarts in 15 minutes
- **Runbook:**
  1. Check Container App logs: `az containerapp logs show`
  2. Look for OOMKilled events (memory limits)
  3. Check health probe configuration
  4. Verify container image is accessible from ACR
  5. Review recent deployments for breaking changes

### Zero Replicas Alert
- **Severity:** 1 (Critical)
- **Threshold:** Running replicas < 1
- **Runbook:**
  1. Check Container App status: `az containerapp show`
  2. Verify min replicas configuration (should be >= 1 for prod)
  3. Check Container Apps Environment health
  4. Review scaling rules and triggers
  5. Manually scale up: `az containerapp update --min-replicas 1`

### Response Time Alert
- **Severity:** 2 (Warning)
- **Threshold:** Average response > 2000ms
- **Runbook:**
  1. Check CPU and memory utilization
  2. Review active connections and request patterns
  3. Check downstream service latencies (Key Vault, Storage)
  4. Consider scaling up replicas or CPU/memory limits
  5. Enable distributed tracing for bottleneck analysis

### Key Vault Availability Alert
- **Severity:** 1 (Critical)
- **Threshold:** Availability < 99%
- **Runbook:**
  1. Check Azure Status page for Key Vault service issues
  2. Verify network rules allow Container App access
  3. Check Managed Identity permissions (Key Vault Secrets User)
  4. Test manual access: `az keyvault secret list --vault-name <name>`
  5. Verify Key Vault soft-delete and purge protection status

### Key Vault Latency Alert
- **Severity:** 2 (Warning)
- **Threshold:** API latency > 500ms
- **Runbook:**
  1. Check Key Vault throttling (429 responses)
  2. Implement client-side secret caching
  3. Reduce frequency of Key Vault calls
  4. Verify Key Vault is in same region as application
  5. Check concurrent request volume

### Error Rate Alert
- **Severity:** 2 (Warning)
- **Threshold:** > 10 errors in 15 minutes
- **Runbook:**
  1. Query Log Analytics for error details
  2. Correlate with recent deployments
  3. Check for dependency failures (external APIs, databases)
  4. Review application exception handling
  5. Check for resource exhaustion (connections, memory)

### Slow Queries Alert
- **Severity:** 3 (Informational)
- **Threshold:** > 5 requests exceeding 5 seconds
- **Runbook:**
  1. Identify slow endpoints from logs
  2. Enable request duration tracing
  3. Profile database queries if applicable
  4. Check for N+1 query patterns
  5. Consider adding caching layer
