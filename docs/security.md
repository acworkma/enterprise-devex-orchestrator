## STRIDE Threat Model

| ID | Category | Threat | Mitigation | Residual Risk |
|----|----------|--------|------------|---------------|
| T-001 | Spoofing | Unauthorized access to API endpoints | Managed Identity + RBAC -- no passwords to steal | Low |
| T-002 | Tampering | Modification of secrets or config | Key Vault with soft delete + purge protection | Low |
| T-003 | Repudiation | Untracked changes to infrastructure | Log Analytics diagnostics + Git audit trail | Low |
| T-004 | Information Disclosure | Secrets leaked in logs or code | structlog sanitization + Key Vault references | Low |
| T-005 | Denial of Service | Resource exhaustion attacks | Container App auto-scaling (1-10 replicas) + HTTP concurrency limits | Medium |
| T-006 | Elevation of Privilege | Excessive permissions for service identity | Least-privilege RBAC (Secrets User, AcrPull, Blob Data Contributor) | Low |

## Supply Chain Security

### Container Image Security
- Multi-stage Docker builds minimize attack surface
- Non-root user (`appuser:appuser`) in production container
- Health check built into Dockerfile
- Images stored in private Azure Container Registry

### Dependency Management
- Dependabot monitors pip, GitHub Actions, and Docker dependencies
- CodeQL scans code for vulnerabilities on every PR
- Weekly scheduled scans for zero-day detection

### CI/CD Pipeline Security
- OIDC federation eliminates stored credentials
- Deployment requires manual trigger (workflow_dispatch)
- Bicep templates validated before deployment
- Container images are tagged with commit SHA

## Incident Response

### Credential Compromise
1. Rotate affected Managed Identity (Azure portal or CLI)
2. Purge compromised secrets from Key Vault
3. Review Log Analytics for unauthorized access patterns
4. Deploy fresh container revision

### Container Image Compromise
1. Quarantine the affected image in ACR
2. Roll back to previous Container App revision
3. Run CodeQL scan on the source code
4. Rebuild and redeploy from clean source
